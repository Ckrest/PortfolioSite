Read Aloud Button for ChatGPT

A Chrome extension that adds a persistent "Read Aloud" button to ChatGPT's bottom toolbar, making it easy to start audio playback without scrolling to the bottom or opening context menus.

Chrome Extension JavaScript UX Design
View on Chrome Web Store →

Problem

ChatGPT's native "Read Aloud" feature is buried in a context menu, requiring users to scroll to the bottom of long messages and click through menus. This creates friction when users want to quickly listen to responses, especially during longer conversations.

Solution

The extension adds a persistent speaker icon button directly to ChatGPT's bottom toolbar. The button:

  • Stays visible while scrolling through long conversations
  • Automatically triggers ChatGPT's native Read Aloud feature without showing menus
  • Updates its icon to show play/stop states based on audio playback
  • Uses event-driven architecture with audio element listeners for minimal performance impact
  • Matches ChatGPT's native design language for seamless integration

Technical Implementation

Built with vanilla JavaScript and Chrome Extension Manifest V3, the extension:

  • Injects a custom button into ChatGPT's DOM structure
  • Programmatically opens and interacts with ChatGPT's "More actions" menu invisibly
  • Uses MutationObserver to detect audio elements and attach event listeners
  • Implements retry logic to handle ChatGPT's async menu rendering
  • Provides visual feedback through icon state changes (play ↔ stop)

The extension is lightweight, requires no background processes, and works entirely client-side with event-driven architecture.

Impact

  • Published on Chrome Web Store with positive user feedback
  • Eliminates the need to scroll and navigate menus for audio playback
  • Improves accessibility by making Read Aloud functionality more discoverable
  • Zero performance overhead with event-driven design
